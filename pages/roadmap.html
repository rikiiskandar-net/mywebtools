ARSITEKTUR MULTI-PAGE (MPA)

Peta Jalan Aplikasi

Dokumen ini melacak modul yang telah diselesaikan dan rencana pengembangan masa depan.

Infrastruktur utama aplikasi web telah berhasil dibangun dan diisolasi ke dalam file HTML independen.

  • Struktur Terpisah: Berjalan di atas index.html (Redirector), login.html, register.html, dan dashboard.html.
  • Otentikasi Dasar: Modul Login dan Pendaftaran Supabase sudah beroperasi dengan Route Guards.
  • UI: Menggunakan gaya desain Google Account (Material 3 / Dark Mode).
root/
  • ├── .agents/
  • ├── public/
  • ├── src/
  • ├── pages/
  • │  ├── login, register, dashboard
  • ├── index.html

Direktori Inti

  • .agents/: Instruksi AI Agent.
  • src/js/: Otak logika JavaScript.

Konfigurasi

  • vite.config.js: Mesin kompilasi MPA.
  • vercel.json: Pengatur Clean URLs.

Memisahkan wewenang antara pengguna biasa dan Administrator.

  • Memanfaatkan Metadata Supabase (raw_app_meta_data) untuk menyimpan status role.
  • Memberlakukan guard tambahan: Menu tertentu otomatis menendang keluar pengguna jika mereka bukan admin.

Peningkatan antarmuka untuk kenyamanan dan efisiensi ruang.

  • Navigasi sidebar bisa di-minimize/maximize dengan state tersimpan di localStorage.
  • Implementasi penuh Lucide Icons yang lebih ringan dan modern.
  • Sistem Clean URLs: Semua file .html telah dihilangkan dari baris alamat browser.

Ekspansi aplikasi dengan menambahkan fitur alat produktivitas harian berbasis Vanilla JS.

  • Struktur Grid Layout yang interaktif untuk etalase alat.
  • Implementasi fitur pertama: Password Generator dengan kalkulasi kekuatan kata sandi real-time dan opsi mutasi karakter (Leet Speak).

Menghidupkan tabel dokumen di dashboard.html dengan operasi Create, Read, Update, Delete (CRUD).

  • Mengambil data riwayat dokumen dari database Supabase.
  • Mengaktifkan form unggah dokumen via Modal UI.
  • Menentukan privasi (relasi): Publik atau Privat (berbasis user_id).

Melindungi API Supabase dari manipulasi jahat dengan mengunci tabel secara logis di level database.

  • SELECT: Kebijakan baca hanya untuk pengguna sah.
  • INSERT/UPDATE/DELETE: Kebijakan ketat modifikasi data hanya untuk Admin/pemilik data.
