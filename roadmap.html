ARSITEKTUR MULTI-PAGE (MPA)

Peta Jalan Aplikasi

Dokumen ini melacak modul yang telah diselesaikan dan rencana pengembangan masa depan untuk sistem MyWebTools berbasis Multi-Page Application.

Fondasi & Arsitektur MPA

SELESAI

Infrastruktur utama aplikasi web telah berhasil dibangun dan diisolasi ke dalam file HTML independen.

  • Struktur File Terpisah: Berjalan di atas index.html (Redirector), login.html, register.html, dan dashboard.html.
  • Otentikasi Dasar: Modul Login dan Pendaftaran Supabase sudah beroperasi dengan penjaga rute (Route Guards) di setiap halaman.
  • Pengalaman Pengguna (UI): Menggunakan gaya desain Google Account (Material Design 3 / Dark Mode).
1

Logika Peran Akses (RBAC)

MENDATANG

Memisahkan wewenang antara pengguna biasa dan Administrator.

  • Memanfaatkan Metadata Supabase (raw_app_meta_data) untuk menyimpan status role.
  • Memberlakukan guard tambahan: Menu tertentu (seperti manajemen pengguna di users.html) otomatis menendang keluar pengguna jika mereka bukan admin.
2

Manajemen Dokumen Penuh

MENDATANG

Menghidupkan tabel dokumen di dashboard.html dengan operasi Create, Read, Update, Delete (CRUD).

  • Mengambil data riwayat dokumen dari database Supabase.
  • Mengaktifkan form unggah dokumen via Modal UI.
  • Menentukan privasi (relasi): Apakah dokumen bersifat Publik (terlihat semua) atau Privat (berbasis user_id).
3

Keamanan Database (RLS Policies)

MENDATANG

Melindungi API Supabase dari manipulasi jahat dengan mengunci tabel secara logis di level database.

  • SELECT: Kebijakan agar data hanya bisa dibaca oleh pengguna yang telah login secara sah.
  • INSERT/UPDATE/DELETE: Kebijakan ketat bahwa aksi memodifikasi data hanya bisa dilakukan oleh peran (Role) Admin atau pemilik data.
